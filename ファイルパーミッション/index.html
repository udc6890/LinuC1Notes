    ファイルパーミッション

    ファイルパーミッション

    ファイルパーミッションとは、ファイルへのアクセス権限の事。すべてのファイル(ディレクトリ含む)はパーミッションを持ち、パーミッションが割り当てられないユーザはファイルへアクセスできない。

    通常ファイルとスペシャルファイル

    Linuxではファイルは大きく分けて通常ファイルとスペシャルファイルの2種類がある。

    通常ファイル

    .txtなどのようなファイル、ディレクトリ、リンクファイル、ハードリンク、シンボリックリンク、ソフトリンク、実行ファイル。

    スペシャルファイル

    デバイスファイル、パイプ、ソケット。

    ファイル種別の表記

    ls -lを実行した時の行頭でファイルの種類が分かる。

    表示 ファイルの種別
    - ファイル
    d ディレクトリ
    l リンク
    p パイプファイル
    b ブロックデバイス
    c キャラクタデバイス

    権限の制御単位

    パーミッションは3つの範囲で制御できるように分けられており、それぞれの範囲ごとに権限を設定できる。

    • ファイルの所有者であるユーザー (ユーザーパーミッション)
    • 所有権を持つグループ (グループパーミッション)
    • システム上のその他すべてのユーザー (所有権を持つグループのユーザーやメンバー以外) (その他のパーミッション)

    RWXパーミッション

    すべてのファイルには読み取り(Read)、書き込み(Write)、実行可能(Execute)の3つのカテゴリのパーミッションが用意されている。また、それぞれの属性には数字が割り当てられており、パーミッション属性変更コマンドchmodでその数字を指定して実行できる。

    パーミッションに応じてできる事

    パーミッション ファイル ディレクトリ
    R ファイルの内容を参照 ディレクトリ内の内容を参照
    W ファイルの書き込み ディレクトリ内に新規ファイル作成
    既存ファイルの名前変更と削除
    X ファイルの実行 ディレクトリ内へのアクセス
    ディレクトリへの移動

    パーミッションの表示

    ls -lコマンドを実行すると以下のように、スペースで区切られた10桁の英字が表示される。このうち先頭を除いた9桁がパーミッションの表示。

    d rwx rwx r--

    先頭(この場合はd)はファイルの種別を現し、残り9桁は3桁毎に解釈される。

    最初の3桁は所有者のパーミッション、次の3桁は所有グループのパーミッション、最後の3桁はその他のユーザーのパーミッションを意味する。

    ファイル種別 所有者 所有グループ その他
    d rwx rwx rwx

    デフォルトパーミッション

    ファイル・ディレクトリを新規作成するとき、それぞれのパーミッションはファイルが666、ディレクトリが777になっている。

    mask値

    rootの場合は0022、一般ユーザの場合は0002が設定されており、新しくファイル・ディレクトリを作成する場合にこの値をもとにファイルのパーミッションが決められる。mask値は/etc/bashrcで設定されている。

    特殊なパーミッション

    パーミッションにはRWXパーミッションの他に、特殊なパーミッションが存在する。

    SUID:Set User ID

    誰がそのファイルを実行しても、セットされたユーザで実行されるという状態。 SUIDを設定すると、所有者のxパーミッションの表示がsに変更される。もともと実行権限を所有していなかった場合は大文字のSになる。chmodコマンドで4000を追加する

    passwd,wallコマンドなどで使われる。

    SGID:Set Group ID

    誰がそのファイルを実行しても、セットされたグループで実行されるという状態。 ディレクトリにSGIDを指定しておくと、ディレクトリ内部で作成されたファイルは全てディレクトリのSGIDで指定したグループが割り振られる。グループのxパーミッションの表示がsに変更される。もともと実行権限を所有していなかった場合は大文字のSになる。 chmodコマンドで2000を追加する。

    Stickey bit

    ディレクトリに設定される特殊なパーミッション。すべてのユーザが書き込みができるが、所有者かroot以外削除ができない。 他のユーザの実行権限がtになる。もともと実行権限を所有していなかった場合は大文字のTになる。 chmodコマンドで1000を付ける。主に/tmpディレクトリに使用されている。

    パーミッション管理コマンド

    内容 コマンド
    ファイル・ディレクトリのパーミッションを変更する。 chmod
    ファイル・ディレクトリの所有者を変更する chown
    ファイル・ディレクトリの所有グループを変更する chgrp
    ファイル・ディレクトリを作成する際のmask値を変更する umask
